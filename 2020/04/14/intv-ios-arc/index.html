記憶體管理

程式運行中所產生的物件、變數、函式都會佔用到記憶體，而記憶體有限，系統會有個『機制』回收用不到的對象，已清出多的記憶體再重新分配，如果佔用過多記憶體，就會導致閃退、Crash。
ARC跟MRC就是上面所說的機制，差別在ARC是compiler會自動(Auto)幫你做這件事情，MRC需要手動(Manual)用程式去做這件事情。

Reference Counting

Reference count(以下簡稱ref cnt)或usage count是程式語言的一種記憶體管理技術，把資源的引用次數存起來，當ref cnt為0的時候，也就是可以從記憶體中釋放的時候了。
這方法優點是，可以盡快回收不再被使用的資源，缺點就頻繁更新ref count可能會導致運行效率，另外，也可能產生retain cycle問題。

Swift物件的生命週期

在講retain cycle之前，先提一下Swift物件的生命週期，分為五個階段：

  1. 分配記憶體(Allocation)
  2. 執行init(Initialization)
  3. 使用(Usage)
  4. 執行deinit(Deinitialization)
  5. 釋放記憶體(Deallocation)
    上面說的ref count就是在「第三步、使用」這階段，在Swift當中，compiler會幫你 count+1或是count-1，你不用自己DIY，所以就是ARC，早期objc則是使用MRC。
